## 📝 Описание проекта
В данной работе реализован полный цикл жизненного цикла смарт-контрактов: от написания кода на языке **Solidity** до их глубокой верификации. Основная цель — обеспечить 100% корректность логики через **гибридную систему тестирования**.

В ходе работы были решены технические сложности, связанные с сетевыми блокировками (ошибки импорта библиотек Chai через CDN), путем внедрения кастомных инструментов валидации в JavaScript-тестах.

---

## 📂 Структура проекта

### 🔹 contracts/ (Исходный код)
*   `1_Storage.sol` — Базовый контракт для записи и чтения целых чисел.
*   `2_Owner.sol` — Реализация прав владения с использованием модификатора доступа `isOwner`.
*   `3_Ballot.sol` — Система электронного голосования с поддержкой делегирования голосов и подсчетом победителя.
*   `4_VendingMachine.sol` — Торговый автомат, работающий с платежами в Ether (`payable`) и контролем складских запасов.

### 🔹 tests/ (Наборы тестов)
*   **JavaScript (`*.test.js`)**: Интеграционные тесты на базе Ethers.js. Имитируют поведение пользователя или фронтенд-приложения.
*   **Solidity (`*_test.sol`)**: Юнит-тесты, проверяющие взаимодействие контрактов непосредственно внутри блокчейна через плагин **Solidity Unit Testing**.

## 🛠 Инструкция по запуску в Remix IDE

### 1. Компиляция
1. Откройте вкладку **Solidity Compiler** (иконка `S`).
2. Выберите версию компилятора `0.8.2` или выше.
3. Скомпилируйте каждый файл из папки `contracts/`.

### 2. Запуск тестов на JavaScript
1. Перейдите в папку `tests/`.
2. Нажмите **правой кнопкой мыши** на файл (например, `ballot.test.js`).
3. Выберите пункт **Run**. Результаты отобразятся в нижней консоли.

### 3. Запуск тестов на Solidity
1. Откройте плагин **Solidity Unit Testing** (иконка с двойной галочкой).
2. Выберите файлы тестов (например, `Ballot_test.sol`).
3. Нажмите кнопку **Run**. Убедитесь, что все пункты отмечены зеленым цветом (Passed).
